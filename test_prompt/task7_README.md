**负责模块**：模块二 — LLM 抽取 → 知识图谱

## 任务目标

设计一个给 DeepSeek 的指令（Prompt），使其能从教材语料库的一个文本段落中，自动提取出**实体**（概念、算法、工具等）和**关系**（包含、依赖、对比等），并以标准 JSON 格式输出。

## Prompt 设计思路

经过多轮测试和迭代，最终 Prompt 包含以下关键约束：

**约束一：只提取原文明确出现的内容**  
防止 AI 推断或补充原文没有的概念，保证图谱数据可追溯。

**约束二：同义词只保留最常用的一个**  
教材中同一概念往往有多种叫法（如 feature / attribute / predictor），若全部提取会导致实体爆炸，图谱混乱。只保留最通用的那一个，其余忽略。

**约束三：不提取举例用的具体数据**  
书中的表格列名、数值、人名、地名是举例材料，不是知识概念，不应进入图谱。

**约束四：definition 基于原文，不能只翻译名字**  
防止 AI 把 `categorical data` 的定义写成"分类数据"这种无效定义。要求定义包含原文中对该概念的实质解释。

**约束五：明确"对比"关系不适用于同义词**  
同义词之间没有"对比"关系，"对比"只用于两种不同方法或概念的比较。

## 迭代过程

| 版本 | 问题 | 修改内容 |
|------|------|---------|
| v1 | 同义词被当成独立实体，用"对比"关联；提取了表格列名；实体数量爆炸（42个） | 加入规则4、5；补充"对比"的说明 |
| v2 | definition 过于简单，仅翻译实体名称（如"分类数据"） | 要求 definition 基于原文内容，不能只翻译名字 |
| v3（最终版） | 实体精简至合理数量（约8-11个/段），definition 有实质内容 | 当前 `task7_test.py` 中的版本 |

---

## 输出格式

```json
{
  "entities": [
    {
      "id": "e1",
      "name": "实体名称",
      "type": "concept",
      "definition": "基于原文内容的简短定义，不超过50字"
    }
  ],
  "relations": [
    {
      "source": "e1",
      "target": "e2",
      "relation": "包含"
    }
  ]
}
```

**实体类型**：`concept` / `algorithm` / `task` / `process` / `tool`  
**关系类型**：`包含` / `依赖` / `应用于` / `衍生` / `前置条件` / `对比`

---

## 已知局限

- 部分边界模糊的实体（如 `relational databases`）类型归类可能不准确，属于可接受误差
- 若段落内容较少（如版权页、目录），提取结果可能为空或极少，属正常现象
- 该 Prompt 在任务9中会根据任务8的批量试跑结果继续迭代优化
