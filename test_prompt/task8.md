\# 任务8 工作汇报：知识图谱首版抽取与评估



\## 工作概述



基于前序同学（任务7）设计的 LLM 抽取 Prompt，本阶段完成了知识图谱抽取流水线的

工程化落地与批量评估框架的完整搭建，为后续图谱质量优化提供了可量化的评估基础。



\---



\## 一、评估脚本工程化开发



在任务7仅有单条测试代码的基础上，从零设计并实现了完整的批量评估脚本

`08\_run\_and\_evaluate.py`，主要功能包括：



\- \*\*多书批量处理\*\*：支持同时对多本语料库文件进行抽取，本次选取英文书

&#x20; 《Practical Statistics for Data Scientists》与中文书《可解释人工智能导论》

&#x20; 各抽取 15 个 chunk，覆盖中英文双语场景

\- \*\*断点续跑机制\*\*：每处理完一个 chunk 立即写入磁盘，程序中断后重启可

&#x20; 自动跳过已成功的记录，失败记录自动重试，避免重复 API 消耗

\- \*\*双格式输出\*\*：

&#x20; - `eval\_run\_08.json`：保存完整抽取结果，供程序后续使用

&#x20; - `eval\_report\_08.csv`：人工评估表格，每行对应一个实体或关系，

&#x20;   含原文片段、评估打分列、问题描述列，可直接用 Excel 打开填写



\---



\## 二、代码健壮性优化



针对生产运行中发现的问题进行了多项工程改进：



\- \*\*异常分类处理\*\*：区分 JSON 解析失败与 API 调用失败两类错误，分别记录，

&#x20; 不因单条失败中断整个流程

\- \*\*智能续跑\*\*：仅跳过成功记录，失败 chunk 在下次运行时自动重试（修复了初版

&#x20; 将失败记录也纳入跳过集合的 bug）

\- \*\*去重写入\*\*：重试成功时替换旧的失败记录，避免结果文件中出现重复条目



\---



\## 三、评估框架设计



设计了结构化的人工评估维度，生成对应的评估表格：



| 评估维度 | 说明 |

|----------|------|

| 实体名称是否来自原文 | 检验 AI 是否杜撰了原文没有的概念 |

| 实体类型分类是否正确 | concept / algorithm / task / process / tool |

| 定义是否有实质内容 | 排除"仅翻译实体名称"的无效定义 |

| 关系方向是否正确 | source → target 的逻辑是否成立 |

| 关系类型是否准确 | 包含 / 依赖 / 应用于 / 衍生 / 前置条件 / 对比 |

| 是否存在幻觉 | AI 是否推断了原文未明确提及的内容 |
