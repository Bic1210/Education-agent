# 任务9 执行模板

任务9的目标是：根据任务8评估结果，反复迭代 Prompt，并通过固定样本复跑验证质量是否提升。

## 一、输入材料

- Prompt 基线：
  - `task7_test.py`
  - `prompt_v2.py` 或后续版本
- 评估依据：
  - `评估总结.md`
  - `eval_report_08_scored.csv`
- 测试语料：
  - Book 04
  - Book 09
  - 固定 chunk: `5,6,7,8`

## 二、每一轮都要做什么

1. 明确本轮只解决哪些问题
- 不要一轮同时改十几个点
- 每轮最多聚焦 2-4 类高频错误

2. 修改 Prompt
- 只根据已有评估问题改
- 每个改动都要能对应到某类错误

3. 复跑固定样本
- 用同一批 chunk 重新抽取
- 不换样本，保证版本间可比较

4. 做人工评估
- 对比上一轮输出
- 记录新增问题和消失的问题

5. 判断是否进入下一轮
- 如果高频错误下降，就保留当前改动
- 如果没改善，回退思路，重新改 Prompt

## 三、本轮记录格式

每轮都在 `rounds/` 下新建一个文件，命名建议：

```text
round_01_baseline.md
round_02_reduce_redundant_relations.md
round_03_fix_entity_boundary.md
```

每轮记录至少包含：

- 本轮版本名
- 修改目标
- 修改点
- 复跑样本
- 结果观察
- 结论
- 下一轮动作

可以直接复制：
- `templates/round_record_template.md`

## 四、人工评估时重点看什么

实体：
- 名称是否确实出现在原文
- 是否是稳定知识节点，而不是例子/属性/局部细节
- 类型是否正确
- 定义是否有核心区分特征

关系：
- 是否能被原文直接支持
- 是否存在过度解读
- 是否过于冗余
- `包含` 方向是否正确
- 是否把“同义词”误当成“对比”

## 五、建议的达标标准

进入任务10前，至少满足下面这些条件：

1. 实体边界明显更稳
- 评测维度误抽基本消失
- 冗余/过于工具化实体显著减少

2. 关系明显更收敛
- 过度概括、过度解读关系显著减少
- `包含` 方向错误只剩少量个例

3. 定义质量可接受
- 空泛定义明显减少
- 不再频繁出现两个不同概念写出同样定义

4. 中英文都能工作
- Book 04 不再明显失控
- Book 09 保持当前较好水平，不因收紧规则而大量漏抽

## 六、什么时候不该继续改 Prompt

以下问题不该继续靠 Prompt 硬解决：

- 跨 chunk 重复提取
- 多书之间同义词归一
- 全局去重与实体合并

这些应留给任务11处理。
