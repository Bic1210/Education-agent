\## 四、主要问题分析



\### 问题1：跨 chunk 重复（占比最高，30%）

同一概念在多个 chunk 中被反复抽取，如 `histogram`（出现6次）、`random sampling`（出现4次）、`boxplot`（出现3次）。  

\*\*根本原因\*\*：Prompt 没有全局去重机制，每个 chunk 独立处理。  

\*\*改进方向\*\*：在图谱合并阶段增加实体去重逻辑。



\### 问题2：定义空洞或有误（17%）

部分定义形如"与抽样相关的一个概念"、"一种抽样方法"，完全没有实质内容；`variance` 的定义与 `standard deviation` 相同，属于定义混淆。  

\*\*改进方向\*\*：Prompt 中强调"定义必须包含该概念的核心区分特征，不能只是重复名称或泛化描述"。



\### 问题3：评测维度误作节点（8%）

如"客观性"、"简单易懂性"、"互洽性"、"计算效率"被抽取为独立实体，但它们实际上是"可解释性方法"的评测维度，不应作为知识图谱节点。  

\*\*改进方向\*\*：Prompt 中新增规则："不要将某个概念的评测指标、子属性单独抽取为实体节点"。



\### 问题4：实体类型标注错误（12%）

可视化图表（`Boxplots`、`Contour plot`）被标为 `algorithm`；散点图被标为 `tool`；列联表被标为 `tool`。  

\*\*改进方向\*\*：在实体类型定义中补充反例，明确区分"可视化图表 = concept"与"计算工具 = tool"。



\### 问题5：关系方向/类型错误（20%合计）

常见错误：把"包含"的方向搞反（如 `binary variable → categorical data`）；把"是一种"（包含）误标为"衍生"（如 Spearman's rho → correlation coefficient）。  

\*\*改进方向\*\*：在关系定义中增加示例，强调"A 包含 B = B 是 A 的子类"，并要求 AI 在输出前检查方向。

\## 五、Prompt 优化建议（优先级排序）



| 优先级 | 建议 |

|---|---|

| 🔴 高 | 禁止将评测指标/子属性单独抽取为实体节点 |

| 🔴 高 | 关系"包含"需明确方向：父类在 source，子类在 target |

| 🟡 中 | 定义必须包含核心区分特征，不能是空泛描述 |

| 🟡 中 | 区分 concept（可视化图表）与 tool（软件/框架）的边界 |

| 🟢 低 | 合并阶段处理跨 chunk 重复，不需要修改 Prompt |
